# 算法题解：第 K 小回文子串得分计算 (Palindromic Substring)

## 一、 题目概述
给定一个长度为 $n$（$n \le 10^5$）的字符串，王国中有 $m$ 个人（$m \le 20$），每个人对 26 个小写字母有不同的 26 进制赋值（价格）。
对于每一个回文子串，其得分为：**截取该回文串的前半部分（奇数长度则丢弃正中心字符），按该人的字母价格计算 26 进制值，并对 $777777777$ 取模**。
**目标**：对每一个人，求出在所有回文子串（包含重复出现）中，得分第 $K$ 小的得分是多少。

---

## 二、 核心难点与“破局三步曲”

这道题是经典的“缝合怪”，直接暴力破解会面临三重绝境：
1. **时间爆炸 (TLE)**：子串数量高达 $O(n^2)$，每个人算分又要遍历字符串。
2. **空间爆炸 (MLE)**：为了加速算分预处理巨大的二维前缀和数组（$10^5 \times 26$），在严格内存限制下直接撑爆。
3. **GC 爆炸 (TLE/MLE)**：Java 在大数量级下频繁 `new` 对象会导致垃圾回收器崩溃。

### 🔪 第一步：数据结构降维 —— 回文自动机 (PAM)
利用**回文自动机**的性质：一个长度为 $n$ 的字符串，其**本质不同的回文子串最多只有 $n$ 个**。
* **建树**：通过 `fail` 指针（寻找最长回文后缀）在 $O(n)$ 时间内揪出所有本质不同的回文串。
* **计数**：建树完成后，从叶子节点顺着 `fail` 指针向根倒推（`cnt[fail[i]] += cnt[i]`），即可在 $O(n)$ 内统计出每种回文串在原串中的真实出现次数。

#### 性质的解释
为了理解这个结论，我们不要去看整个字符串，而是换一个动态的视角：想象你从左到右，一个字符一个字符地构建这个字符串。

这里有一个核心定理：每次你在末尾增加一个新字符，最多只能产生 1 个“以前从未出现过”的新回文串。

如果这个定理成立，那么你加了 n 次字符，新回文串的数量最多也就是 n 个。对吧？

那为什么每次加一个字符，最多只能产生 1 个新的回文串呢？

我们可以用反证法，用大白话来推演一下：

1. 假设你现在加入了一个新字符，导致在字符串末尾一下子诞生了两个“以前从未见过”的全新回文串。

2. 我们叫长一点的那个回文串为 大回文，短一点的叫 小回文。

3. 注意，这两个回文串都是以你刚加的这个新字符结尾的。所以，小回文 一定是 大回文 的“尾巴”（后缀）。

4. 最关键的一步来了： 因为 大回文 本身是一个回文串，它是左右对称的！既然 小回文 是它的尾巴，那么根据镜面对称的原理，大回文 的“头部”（前缀）一定也长得跟 小回文 一模一样。

5. 既然 小回文 出现在了 大回文 的头部，说明这个结构在前面早就已经被你遍历过、出现过了！

6. 矛盾了！ 我们一开始假设“小回文是以前从未见过的新回文串”，但推导发现它其实早就存在了。

结论：

每次加一个新字符，哪怕它在末尾凑成了好几个回文串，这里面最多只有一个是最长的那个是全新的，其余较短的肯定在前面已经出现过了。

### 🔪 第二步：数学逻辑降维 —— $O(1)$ 树形 DP 算分 (全景剖析)

在放弃了占用 20MB 内存的“26维前缀和”数组后，我们要如何在不遍历字符串的情况下，瞬间算出一个回文串的得分？

答案就藏在 PAM（回文自动机）的生长结构和 26 进制的运算规律里。我们分三层逻辑来推导：

#### 1. 重新审视“截断法则”与半长 $H$
题目要求：将回文串截断，扔掉后半部分（如果长度是奇数，正中间的字符**保留**，扔掉中间字符后面的部分）。
我们观察一下截断后的长度规律：
* 长度 4 (`abba`) $\rightarrow$ 保留 `ab`，长度 2。
* 长度 3 (`aba`) $\rightarrow$ 保留 `ab`，长度 2。
* 长度 7 (`abacaba`) $\rightarrow$ 保留 `abac`，长度 4。

可以得出一个完美的统一公式：如果原回文串长度为 $L$，截断后保留的长度 $H$ 为：
$$H = \lfloor \frac{L + 1}{2} \rfloor$$
*(在 Java/C++ 中，直接写成 `(L + 1) / 2` 即可)*

#### 2. PAM 的隐藏“基因遗传”规律（视觉推导）
PAM 的建树规则是：新回文串 $np$，是在旧回文串 $u$ 的左右两端，包上一层新外壳字符 $c$ 形成的。
那么，当我们把 $np$ 和 $u$ 都按题目的规则**砍掉一半**，它们的前半截长什么样？

我们拿 $u = \text{"ccbcc"}$，外加一层外壳 $c = \text{"a"}$ 举例：

> **旧套娃 $u$**：`c c b c c` (长度 5)
> 砍掉一半后保留的字符串：**`c c b`** (长度 3)
>
> **新套娃 $np$**：`a c c b c c a` (长度 7)
> 砍掉一半后保留的字符串：**`a c c b`** (长度 4)

**发现神奇的等式了吗？**
`a c c b` 完全等于 `a` 拼上 `c c b`！
也就是说：**新套娃的前半截 $=$ 新外壳字符 $+$ 旧套娃的前半截**。
由于 PAM 里的所有节点都是这么一层层包出来的，所以这个拼接规律在整棵回文树上 $100\%$ 适用。

#### 3. 26 进制得分的 $O(1)$ 转移公式
既然字符串是完美拼接的，那 26 进制的得分怎么算？

假设某个人的价格表里，字符 $c$ 的价格是 $v_c$。
截断后的字符串是按 26 进制计算的。当我们把字符 $c$ 拼在一个已有字符串的最前面时，相当于给这个 26 进制数**增加了一个最高位**。

举个十进制的简单例子来类比：
如果你有一个数字 `23`，你想在前面拼一个 `7` 变成 `723`。你怎么用数学公式算？
`723 = 7 * 100 + 23`，也就是 `新数字 = 新数字的值 * 10^(总长度-1) + 老数字`。

换到这道题的 26 进制里，逻辑一模一样：
* 已知旧套娃 $u$ 的得分为：$Score_u$
* 新套娃 $np$ 截断后的总长度为：$H$
* 新外壳字符 $c$ 带来的增量为：$v_c \times 26^{H - 1}$

所以，新节点的得分可以直接由它爹的得分推导出来：
$$Score_{np} = \left( Score_u + v_c \times 26^{H - 1} \right) \pmod{777777777}$$

#### 4. 为什么这是终极降维打击？
理解了这个公式，你再看回文自动机的节点编号。
建树时，新节点的编号 $np$ 永远是 $++sz$，严格大于它爹 $u$ 的编号。
这就构成了一个完美的**拓扑序**！

当有顾客带着他的价格表来查询时：
1. 我们**不需要**去原字符串里截取字符。
2. 我们**不需要**用二维数组做前缀和相减。
3. 我们只需要一个普通的 `for` 循环，从节点 `2` 遍历到 `sz`。
4. 对每一个节点，查一下它爹是谁（`pam_parent[i]`），当初包的什么外壳（`pam_edge[i]`）。
5. 直接用上面的公式，**做一次加法、一次乘法、一次取模**，当前节点的得分就求出来了！

**复杂度对比：**
* 之前的方法：20 人 $\times$ 10万节点 $\times$ 26 个字母循环 $= 5200$ 万次取模运算 $\rightarrow$ **必定 TLE（超时）**。
* 树形 DP 方法：20 人 $\times$ 10万节点 $\times$ 1 次 $O(1)$ 转移 $= 200$ 万次基础运算 $\rightarrow$ **速度起飞，毫秒级 AC，且彻底干掉了 20MB 的巨大二维数组！**

### 🔪 第三步：工程性能降维 —— 64位运算“打包”打败 GC
彻底抛弃 `class Node implements Comparable`。
在 Java 中，几百万次的对象分配会引发严重的性能灾难。
* **压缩**：得分最大为 $777777777$（占 30 位），出现次数最大为 $100000$（占 17 位）。用一个原生的 `long` 型变量，**高 32 位存得分，低 32 位存次数**：
  `pack[i] = (score << 32) | (cnt & 0xFFFFFFFFL)`
* **排序**：直接对原生 `long[]` 数组调用 `Arrays.sort`，实现 **0 内存分配、极致缓存命中** 的工业级排序。

## 三、 核心代码模板 (Java)

```java
import java.io.*;
import java.util.*;

public class PalindromicSubstring {
    public static void main(String[] args) {
        new Solution().run();
    }
}

class Solution {
    Kattio io;
    int maxn = 100005;
    int mod = 777777777;
    
    // PAM 核心数组
    int[] len = new int[maxn];
    int[] fail = new int[maxn];
    int[][] ch = new int[maxn][26];
    int[] cnt = new int[maxn];
    
    // 树形 DP 专用：记录套娃层级关系
    int[] pam_parent = new int[maxn]; 
    int[] pam_edge = new int[maxn];   
    
    // 辅助与状态数组
    int[] pow26 = new int[maxn];
    long[] nodeScore = new long[maxn]; 
    long[] pack = new long[maxn];      // 替代 Node 对象的原生打包数组
    int last, sz;

    void run() {
        io = new Kattio();
        solve();
        io.flush();
    }

    void solve() {
        int t = io.nextInt();
        while (t-- > 0) {
            int n = io.nextInt(), m = io.nextInt();
            char[] c = ("#" + io.next()).toCharArray();
            
            // 1. 预处理 26 的次幂 (防溢出)
            pow26[0] = 1;
            for (int i = 1; i <= n; i++) {
                pow26[i] = (int)((pow26[i-1] * 26L) % mod);
            }
            
            // 2. 初始化并构建 PAM
            init();
            for(int i = 1; i < c.length; i++) insert(i, c);
            
            // 3. 倒推补全真实出现频次
            for (int i = sz; i >= 2; i--) cnt[fail[i]] += cnt[i];
            
            // 4. 响应查询
            while (m-- > 0) {
                long k = io.nextLong();
                int[] w = io.nextIntArray(26); // 当前人的字母价格
                
                int packSz = 0;
                nodeScore[0] = 0; nodeScore[1] = 0;
                
                // 核心：利用拓扑序 O(1) 继承父亲的得分
                for (int i = 2; i <= sz; i++) {
                    int p = pam_parent[i];       // 内部包裹的旧套娃
                    int edgeChar = pam_edge[i];  // 新包的外壳字母
                    int H = (len[i] + 1) / 2;    // 当前回文串一半的长度
                    
                    // 当前得分 = 旧套娃得分 + 新外壳字母贡献的最高位得分
                    long weight = ((long)w[edgeChar] * pow26[H - 1]) % mod;
                    nodeScore[i] = (nodeScore[p] + weight) % mod;
                    
                    // 神仙操作：高32位存得分，低32位存次数
                    pack[packSz++] = (nodeScore[i] << 32) | (cnt[i] & 0xFFFFFFFFL);
                }
                
                // 极速排序
                Arrays.sort(pack, 0, packSz);
                
                // 寻找第 K 小
                long sum = 0;
                for (int i = 0; i < packSz; i++) {
                    sum += (pack[i] & 0xFFFFFFFFL); // 取低32位(次数)
                    if (sum >= k) {
                        io.println(pack[i] >>> 32); // 取高32位(得分)无符号右移
                        break;
                    }
                }
            }
            io.println(); // 每个 Case 后要求空行
        }
    }

    // 初始化 PAM 的奇偶双根
    void init() {
        sz = 1; last = 0;
        len[0] = 0; fail[0] = 1; cnt[0] = 0;
        Arrays.fill(ch[0], 0);
        len[1] = -1; fail[1] = 1; cnt[1] = 0;
        Arrays.fill(ch[1], 0);
    }

    // 寻找能包裹新字符的最长回文后缀 (找备胎)
    int get_fail(int u, int i, char[] s) {
        while (s[i - len[u] - 1] != s[i]) u = fail[u];
        return u;
    }
    
    // 在 PAM 中插入字符
    void insert(int i, char[] s) {
        int c = s[i] - 'a';
        int u = get_fail(last, i, s);
        
        if (ch[u][c] == 0) { // 发现全新回文串
            int np = ++sz;
            Arrays.fill(ch[np], 0); // 清除历史多测脏数据
            cnt[np] = 0;
            len[np] = len[u] + 2;
            
            // 记录树形层级，用于后续 O(1) 算分
            pam_parent[np] = u;
            pam_edge[np] = c;
            
            int fail_node = get_fail(fail[u], i, s);
            fail[np] = ch[fail_node][c]; // 认领备胎
            ch[u][c] = np;
        }
        last = ch[u][c];
        cnt[last]++; // 当前最长回文串出现次数 +1
    }
}
// (注：需配合 Fast I/O 模板 Kattio 使用，此处略去通用 IO 代码)
```

## 四、 复杂度总结
* **时间复杂度**：
  * 建树与统计：$O(n)$。
  * 单次查询：$O(|sz| \log |sz|)$，其中 $|sz| \le n$ 为本质不同的回文串种类数。因为采用原生类型排序，常数极小。总时间 $O(n + m \cdot |sz| \log |sz|)$，轻松跑进时限。
* **空间复杂度**：$O(n \times \Sigma)$，其中 $\Sigma = 26$（字符集大小）。因彻底摒弃了巨型预处理数组和对象开销，峰值内存占用被压缩在 **15MB** 以内。
